React Forms

<input>, <textarea>, and <option> differ from other native components.

Mutated via user interactions.

Provide interfaces to manage forms in response to user interactions.

Interactive Props

Props affected via user interactions:

  • value: supported by input and textarea.
  • checked: supported checkbox or radio.
  • selected: supported by option.

Form components allow listening for changes.

Set a callback to the onChange prop.

Callback fires when:

  • The value changes.
  • Checked state changes.
  • Selected state changes.

Controlled Components

Any <input> with value set is a controlled component.

Value of the rendered element will always reflect the value prop.

Uncontrolled Components

Any <input> that does not supply a value (or sets it to null).

Value of the rendered element will reflect the user's input.

Validation

We can now use controlled components in our form to enforce validation.


<form name="employeeForm">
  <input name="username" 
    value={this.props.employee.username}
    onChange={this.props.validate} />
</form>




validate: function (event) {
  var field = event.target.name;
  var value = event.target.value;
  
  // min length 1
  if (!value || value.length < 1) {
    this.state.errors[field] = 
      'You must provide a username.';
  }
  // max length 40
  else if (value.length > 40) {
    this.state.errors[field] = 
      'Username can only be 40 characters long.';
  }
  
  this.props.employee[field] = value;
  return this.setState({
    employee: this.state.employee, 
    errors: this.state.errors
  });
},
